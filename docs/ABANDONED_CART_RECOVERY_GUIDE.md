# Abandoned Cart Recovery Guide

## Overview

The Abandoned Cart Recovery system tracks shopping carts that customers start but don't complete, sends recovery emails, and fires CRM alerts to customers — all tenant-scoped and fully integrated with the platform's CRM customer portal.

## Architecture

```
Customer (browser)
  │
  ├─ cartManager.ts → saveCart() → debounced 5s → POST /api/cart/track
  │                                                        │
  │                                                        ▼
  │                                          AbandonedCartService.trackCart()
  │                                                        │
  │                                               abandoned_carts table
  │                                                        │
  │  Scheduled Job (every 30 min)                          │
  │  abandoned-cart-recovery.ts                            │
  │    └─ getCartsForRecovery(1h–24h window)               │
  │    └─ sendRecoveryEmail(cartId)                        │
  │         ├─ emailService.sendEmail()  → customer email  │
  │         └─ CrmAlertService.createAbandonedCartAlert()  │
  │                    │                                   │
  │                    ▼                                   │
  │              crm_alerts table (type: 'abandoned_cart') │
  │                    │                                   │
  │  Customer CRM Portal                                  │
  │  GET /api/customer/crm/alerts                          │
  │    └─ filtered by metadata.customer_email              │
  │                                                        │
  └─ Checkout success                                      │
       └─ markCartConvertedByEmail()                       │
            └─ sets converted=true, converted_at, order_id │
```

## Components

### 1. Database — `abandoned_carts` table

- **Migration:** `database/migrations/055_abandoned_carts.sql`
- **Prisma model:** `abandoned_carts` in `apps/api/prisma/schema.prisma`
- **ID format:** `acart-{tenantKey}-{nanoid}` (tenant-scoped, generated by `generateAbandonedCartId`)
- **Key columns:**
  - `cart_id` — localStorage cart key from frontend
  - `customer_email` — used for recovery email + CRM alert ownership
  - `items` — JSONB array of cart items (product_id, name, qty, price, image, variant)
  - `cart_value_cents` — total cart value
  - `recovery_email_sent` / `recovery_email_sent_at` — tracks email state
  - `converted` / `converted_at` / `converted_order_id` — set on checkout

### 2. Backend Service — `AbandonedCartService.ts`

**File:** `apps/api/src/services/AbandonedCartService.ts`

Singleton extending `BaseService`. Key methods:

| Method | Purpose |
|--------|---------|
| `trackCart(input)` | Creates or updates an abandoned cart record. Called from `POST /api/cart/track`. |
| `markCartConverted(cartId, orderId?)` | Marks cart as converted by cart_id. |
| `markCartConvertedByEmail(email, tenantId, orderId?)` | Marks cart as converted by customer email. Called on checkout success. |
| `getAbandonedCarts(tenantId, options)` | Paginated list with filters (converted, recoveryEmailSent). |
| `getSummary(tenantId)` | Returns total, recovered, pending, conversion rate, value stats. |
| `sendRecoveryEmail(cartId)` | Sends recovery email + fires CRM alert. Guards against converted/already-sent/no-email. |
| `getCartsForRecovery(minAge, maxAge)` | Finds carts in the 1–24h window eligible for recovery. |
| `resetRecoveryEmailState(cartId)` | Resets email state so merchant can resend from dashboard. |

### 3. API Routes

#### Cart tracking (public, no auth)
- `POST /api/cart/track` — `apps/api/src/routes/cart.ts`
  - Body: `{ tenantId, cartId, customerEmail?, customerName?, customerId?, items[] }`
  - Called by frontend `cartManager.ts` with 5-second debounce

#### Merchant dashboard (auth required)
- `GET /api/tenants/:tenantId/abandoned-carts` — list with filters
- `GET /api/tenants/:tenantId/abandoned-carts/summary` — summary stats
- `POST /api/tenants/:tenantId/abandoned-carts/:cartId/resend` — resend recovery email

**File:** `apps/api/src/routes/abandoned-carts.ts`

#### Customer CRM portal (customer JWT auth)
- `GET /api/customer/crm/alerts` — includes `abandoned_cart` type alerts, scoped by `metadata.customer_email`
- `PUT /api/customer/crm/alerts/:alertId/read` — mark read (with email ownership check)
- `PUT /api/customer/crm/alerts/:alertId/dismiss` — dismiss (with email ownership check)

**File:** `apps/api/src/routes/crm/customer/crm-customer.ts`

### 4. Scheduled Job — `abandoned-cart-recovery.ts`

**File:** `apps/api/src/jobs/abandoned-cart-recovery.ts`

- Runs every **30 minutes** (initial delay: 1 min after server start)
- Finds carts abandoned **1–24 hours ago** that haven't had a recovery email sent
- Sends recovery emails with 2-second delay between each (rate limit protection)
- Wired into server startup in `apps/api/src/index.ts`

### 5. CRM Customer Alert Integration

When a recovery email is sent, `AbandonedCartService.sendRecoveryEmail()` also fires a CRM alert via `CrmAlertService.createAbandonedCartAlert()`:

- **Alert type:** `abandoned_cart`
- **Alert icon:** 🛒
- **Title:** "Your cart at $XX.XX is waiting for you"
- **Body:** "You left N items in your cart. Complete your purchase before it's gone!"
- **Metadata:** `abandoned_cart_id`, `customer_email`, `cart_value_cents`, `item_count`, `recovery_url`

The customer sees this alert in their CRM support portal at `/api/customer/crm/alerts`. The alert is scoped to the customer by `metadata.customer_email` — only the customer who owns the cart can see, read, and dismiss it.

**Customer alert scope** (`getCustomerAlertScope` in `crm-customer.ts`) includes:
- Tenants where the customer has placed orders
- Tenants where the customer has a `customer_tenant_relationships` record (covers cart abandoners who haven't ordered yet)

### 6. Frontend — Merchant Dashboard

- **Page:** `apps/web/src/app/t/[tenantId]/abandoned-carts/page.tsx`
- **Client:** `apps/web/src/app/t/[tenantId]/abandoned-carts/AbandonedCartsClient.tsx`
- **Service:** `apps/web/src/services/AbandonedCartService.ts` (extends `TenantApiSingleton`)

Features:
- Summary stats (total, recovered, pending, conversion rate, value)
- Filterable table (all / pending / email_sent / converted)
- Search by customer email or name
- Pagination
- Resend recovery email button per cart

### 7. Frontend — Cart Tracking

**File:** `apps/web/src/lib/cart/cartManager.ts`

- `trackCartWithServer()` — sends cart data to `/api/cart/track`
- Called via `saveCart()` with a **5-second debounce** to avoid excessive API calls
- Reads `customer_email`, `customer_name`, `customer_id` from `localStorage`
- Silent failure — tracking is best-effort, never blocks cart operations

## Data Flow Summary

### Cart Tracking
1. Customer adds/removes items in storefront
2. `saveCart()` persists to `localStorage` + debounced server tracking
3. `POST /api/cart/track` → `AbandonedCartService.trackCart()` → upsert in `abandoned_carts`

### Recovery Email + CRM Alert
1. Scheduled job finds eligible carts (1–24h old, not converted, not emailed, has email)
2. `sendRecoveryEmail()` sends HTML email with item list + recovery URL
3. `CrmAlertService.createAbandonedCartAlert()` fires CRM alert to customer
4. `recovery_email_sent` flag set to prevent duplicates

### Checkout Conversion
1. Customer completes checkout
2. `markCartConvertedByEmail()` called (async, non-blocking) from `checkout.ts`
3. Sets `converted=true`, `converted_at`, `converted_order_id`
4. Cart no longer eligible for recovery emails

### Merchant Resend
1. Merchant clicks "Resend" on dashboard
2. `resetRecoveryEmailState()` clears the sent flag
3. `sendRecoveryEmail()` sends a new email + fires a new CRM alert

## Tenant/Admin Connection

- **Tenant (merchant):** Full dashboard access — view all abandoned carts, summary stats, resend emails
- **Platform admin:** No direct admin panel for abandoned carts (tenant-scoped feature)
- **Customer:** Sees CRM alerts in their support portal, can read/dismiss, can create support tickets about abandoned carts
